Hardware interfaces

The most important hardware Shopline requires is a smartphone with a built in camera. The device should be able to activate the camera from within the app to make the sceneform 3D view.The secondary requirement is an active Internet connection, either by mobile data or WiFi.This is to communicate with the Google Firebase Server and to make API calls to the google Sceneform

Software interfaces

Shopline app will run Android 7.0 or later devies which suppors Google Play Services for AR, while a browser-based web interface will run on any computer (Windows, Mac or Linux) with a capable modern web browser. Google’s ArCore Platform will be used to take the advantage of its Sceform based 3D augmented reality functions.We will be using the free versions of the products,for this project purposes.

Communication interfaces

The application will set 3D image to the google ARCore Sceneform.ARCore will do two things: tracking the position of the mobile device as it moves, and building its own understanding of the real world.ARCore's motion tracking technology uses the phone's camera to identify interesting points, called features, and tracks how those points move over time. With a combination of the movement of these points and readings from the phone's inertial sensors, ARCore determines both the position and orientation of the phone as it moves through space.

Other Non-functional Requirements

Performance Requirements

The system will be able to accommodate a large number of items and users. Also, in every action-response of the system, the delay will not be longer than a few seconds.

Safety Requirements

All the data users' need to store, for the future, is always safe with our cloud storage. Even if the user device is broken,through his/her account the data will always remain protected.

Security Requirements

The system will use a secure database. Users can go through basic information but will not be able to modify any information except their personal ones. Also, each user of the system shall login using some form of unique identification (e.g., username and password). All login attempts shall be done so in a secure manner (e.g., encrypted passwords)

Software Quality and Attributes

1. Availability: The system shall be available at all times, that is, the user can access it using a web browser or mobile app, only restricted by the down time of the server on which the system runs. Also in case of a hardware failure, backups of the database can be retrieved from the server and saved by the administrator. Then the service will be restarted.

2. Portability: The end-user part is fully portable and any system using any web browser should be able to use the features of our system. The web part of the system can be used on Windows or Mac etc and the mobile app version can run on Android.

3. Usability: The system is easy to handle and navigates without much delay, and so, the system program reacts accordingly and transverses quickly between its states.

4. Graphical User Interface: The system shall provide a uniform look and feel between all the web pages, along with digital image for each of the products.

Business Rules

The system requires the user to have an account (could be accessed via facebook or google account as well) in order to shop. The user has to be logged in as well. The system will have the option for the users to pay upon delivery of the item.
